Job Market Multi‑Agent Analysis

A CrewAI workflow that collects real job postings, extracts market requirements, compares them to a resume, and generates personalized upskilling recommendations—ending with a polished HTML report.

Who it’s for
Job applicants who want a realistic view of what employers ask for—and what to learn next.
What it delivers
Top in‑demand skills, cloud/platform trends, and a ranked “next skills” plan.
Output artifact
A shareable, visually appealing HTML report + plus JSON/MD traces.

Why this is not an Interactive App?

This workflow takes ~10 minutes for end‑to‑end excution, so this page focuses on the architecture and a real sample report instead of an interactive demo.

Designed for depth over latency
A complete run includes: job collection → requirement extraction → aggregation & normalization → resume parsing → gap analysis → HTML report generation. That end‑to‑end workflow is intentionally batch-style and takes around ~10 minutes (depending on sample size and API speed).

This page therefore focuses on the architecture and a real sample report you can inspect instantly.

Crew architecture

The workflow is a sequential CrewAI process. Each agent owns a single responsibility and passes structured context to the next task.

Each box maps to a CrewAI Agent and its corresponding Task in build_crew.py.

How it works (deeper explanation)

From raw postings to a focused learning plan—in six deterministic steps.

  1. Collect postings for target titles/country via SerpAPI Google Jobs (structured title/company/location/description).
  2. Extract requirements from each description into a strict schema: technical_skills, cloud_platforms, certifications.
  3. Aggregate & normalize across roles: synonym grouping, per-post deduplication, and frequency stats (% of postings).
  4. Parse resume (PDF → text) and extract existing skills/certs/cloud to build an “as‑is” profile.
  5. Gap analysis: compare “market needs” vs. “resume skills” and rank the next 5 highest‑impact learning targets.
  6. Generate an HTML report: charts/tables + a readable narrative, saved as a standalone artifact.
